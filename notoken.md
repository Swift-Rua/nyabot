# Nyabot V2（Zero Token Edition）开发计划书

## —— 无 AI、无 API、无 Token 的拟人化群成员系统

---

# 一、项目目标

## 核心理念

Nyabot 不是聊天 AI，而是一个能够长期混群、学习群文化、形成群人格的数字群友（Pseudo Human）。

整个项目遵循以下原则：

* 不调用任何 AI API
* 不消耗任何 Token
* 不依赖任何大模型
* 所有行为均由统计、规则、概率、模板、历史数据驱动

目标不是回答正确，而是：

> **让群友越来越觉得："它一直生活在这个群里。"**

## Module 1：消息记录器（Logger）

### 功能

记录所有群消息。

每条消息至少包含：

```

nickname
```

### 保存方式

要求：

* 支持快速检索
* 永久保存
* 自动清理超过设定数量的数据

建议：

# 七、设计原则

1. **行为优先于内容**：像真人更多取决于何时说、说多少，而不是说得多聪明。
2. **学习群文化，而不是学习世界知识**：机器人的知识边界就是这个群。
3. **允许犯错和不稳定**：偶尔接不上话、偶尔沉默，比永远完美更像真人。
4. **可解释、可配置**：所有规则、概率、模板均可配置，不依赖黑盒模型。
5. **模块化设计**：每个模块独立，便于逐步迭代或未来替换为更高级的实现（例如接入本地模型或在线 AI）。

---

# 八、未来扩展（保持兼容）

当前架构保持纯离线运行，但所有模块接口应预留扩展能力。例如：

* 本地 Embedding 检索（无需修改消息记录器）
* 本地 LLM 回复生成（无需修改状态机）
* 在线 API 回复（仅替换回复生成器）
* 多人格模式（共享学习数据库，不同状态参数）
* Web 管理面板（查看统计、调整规则）

要求：未来任何 AI 能力都作为**可插拔模块**加入，不破坏 Zero Token Edition 的基础架构。
